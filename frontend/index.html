How it works

Prompt → backdrop → live stage

The product idea: real-time entertainment, not flat images. Type a world, the theatre stages it — generated backdrop, a 2D host reacting to the setting, and procedural particle VFX driven by scene analysis. It's the 2.5D / VFX sweet spot between plain text-to-image and full 3D asset generation.

1

Prompt

User types a world. Client POSTs to /api/scene.

2

Backdrop

Image model (FLUX-class) generates the still plate.

3

Scene read

Server classifies scene type, time of day, mood, palette. (Depth-Anything-V2 slot.)

4

Stage

Browser renders VFX: water, foam, mist, rain, fire, stars, parallax.

5

Host

2D avatar idles, breathes, blinks, gestures, reacts to each scene.

Roadmap ladder

Five rungs, one at a time

  1. Prompt → backdrop. FLUX.1-dev via Diffusers. Seed control + negative prompts.
  2. Backdrop → depth / layers. Depth-Anything-V2. RGBD + foreground/background masks.
  3. Browser stage. Particles, flow, mist, foam, distortion, lighting, parallax. This page, scaled up to WebGPU.
  4. Host avatar. Reactive 2D / 3D character that responds to scene and voice input.
  5. Motion mode. CogVideoX image-to-video for 6-second premium clips.
  6. Later: Gaussian splats (gsplat) or textured 3D assets (Hunyuan3D-2, TRELLIS).

Not a 3D asset generator. A living stage. That's the product wedge.
